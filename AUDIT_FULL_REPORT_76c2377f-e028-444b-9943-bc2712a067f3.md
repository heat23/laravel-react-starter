## Critical Findings

### SEC-CRIT-001: Feature flag global override not restricted to super admin
**File:** `app/Http/Controllers/Admin/AdminFeatureFlagController.php:39-59`

`updateGlobal()` accepts any admin user. Regular admins can toggle billing off, disable api_tokens, or modify any global flag. The FeatureFlagService blacklists only the `admin` flag but not billing/api_tokens/social_auth.

**Impact:** Regular admins can disrupt product availability and revenue.
**Fix:** Add `abort_unless($request->user()?->isSuperAdmin(), 403)` before global override mutations.

---

### SEC-CRIT-002: Webhook replay protection disabled for non-Stripe providers
**File:** `app/Http/Middleware/VerifyWebhookSignature.php:47-53`

Timestamp validation only applies to Stripe. For GitHub/custom providers, `$timestamp` is null and replay protection is bypassed entirely. No idempotency check beyond basic timestamp tolerance.

**Impact:** GitHub/custom webhooks can be replayed indefinitely without detection.
**Fix:** Enforce nonce/request-ID tracking for all providers. Reject webhooks without timestamp or delivery ID.

---

### SEC-CRIT-003: Unbounded admin dashboard queries
**File:** `app/Http/Controllers/Admin/AdminWebhooksController.php:55-88`

Multiple admin endpoints perform `COUNT(*)` on entire tables with no WHERE clause on timeframe. An attacker triggering webhook spam could cause admin dashboard OOM.

**Impact:** DoS via admin dashboard resource exhaustion.
**Fix:** Add `->where('created_at', '>=', now()->subDays(30))` and aggressive caching.

---

### SEC-CRIT-004: dangerouslySetInnerHTML without DOMPurify in JSON-LD
**Files:** `resources/js/Components/seo/BreadcrumbJsonLd.tsx`, `FaqJsonLd.tsx`, + 9 guide/compare pages

Multiple components use `dangerouslySetInnerHTML` with only basic `</script>` escaping. Comments explicitly state "DOMPurify omitted". While content is currently hardcoded, this is inconsistent with Blog/Pricing pages that use DOMPurify and creates risk if content becomes dynamic.

**Impact:** XSS vector if content ever becomes user/CMS-driven. Maintenance risk.
**Fix:** Standardize all JSON-LD injection through DOMPurify.sanitize().

---

### TEST-CRIT-005: 59/63 Form Requests untested
**File:** `app/Http/Requests/`

Only LoginRequest, RegisterRequest, ProfileUpdateRequest, and UpdateSettingRequest have tests. 59 validation rule sets are untested including billing, admin, webhook, and settings requests.

**Impact:** Invalid input could bypass validation silently in production.
**Fix:** Add validation tests for all Form Requests, prioritizing billing and admin.

---

### TEST-CRIT-006: Zero test coverage for background jobs
**File:** `app/Jobs/`

5 background jobs (PersistAuditLog, DispatchWebhookJob, DispatchAnalyticsEvent, CancelOrphanedStripeSubscription, etc.) have no tests.

**Impact:** Job failures in production undetected. Webhook delivery, audit logging, and Stripe cleanup could silently fail.
**Fix:** Add unit tests for each job covering success, failure/retry, and edge cases.

## High Severity Findings

### SEC-HIGH-001: Lock timeout too short for billing
`app/Services/BillingService.php:20-23` — 35s Redis lock may auto-release during slow Stripe API calls, allowing duplicate subscriptions. No Stripe idempotency keys.

### SEC-HIGH-002: No GDPR deletion mechanism for audit logs
`app/Services/AuditService.php:111-146` — Audit logs with IP/user-agent retained indefinitely. No user-deletion cascade. PruneAuditLogs command exists but isn't scheduled.

### SEC-HIGH-003: CSV injection in admin exports
`app/Http/Controllers/Admin/AdminBillingController.php:132-165` — Raw database values streamed to CSV without sanitizing control characters (=, +, -, @). No export audit trail.

### SEC-HIGH-004: No rate limiting on billing operations
`app/Http/Controllers/Billing/SubscriptionController.php` — subscribe, cancel, swap, updateQuantity have no throttle middleware. User could flood Stripe API.

### SEC-HIGH-005: SSRF via webhook endpoint URL
`app/Http/Requests/Webhook/CreateWebhookEndpointRequest.php:19` — URL validation allows localhost/private IPs. Attacker could target internal services.

### SEC-HIGH-006: Admin access attempts not rate-limited
`app/Http/Middleware/EnsureIsAdmin.php:19-25` — No rate limiting on failed admin access, enabling rapid probing and audit log bloat.

### PERF-HIGH-007: PlanLimitService 10s cache TTL causes stampedes
`app/Services/PlanLimitService.php:87-94` — Cache expires every 10 seconds. Simultaneous requests all hit database + Stripe during expiry window.

### SEC-HIGH-008: Trial start not wrapped in transaction
`app/Services/PlanLimitService.php:31-56` — Atomic WHERE NULL UPDATE not in DB::transaction(). Audit logging failure leaves trial active but unlogged.

### DEVOPS-HIGH-009: CI coverage and security checks non-blocking
`.github/workflows/ci.yml:154,222,299` — JS coverage, bundle size, and npm audit use continue-on-error: true.

### TEST-HIGH-010: 14/16 console commands untested
`app/Console/Commands/` — Only SendReEngagementEmails and PruneAuditLogs tested. Cron-triggered commands for dunning, webhook pruning, subscriptions uncovered.

### TEST-HIGH-011: 3 core services untested
LeadScoringService, LifecycleService, DataHealthService have zero test coverage.

### SEC-HIGH-012: 3 high-severity npm vulnerabilities
lodash-es (prototype pollution), picomatch (method injection/ReDoS), brace-expansion (memory exhaustion). All fixable via `npm audit fix`.
