## Strengths

1. **Redis-locked billing mutations** -- BillingService wraps all Stripe operations in Redis locks (35s timeout) with DB transactions, preventing race conditions
2. **Comprehensive admin panel** -- 25 controllers with audit logging, cache invalidation, role-based access (admin/super_admin)
3. **DOMPurify on user-facing HTML** -- Blog content and 2FA QR codes use DOMPurify with explicit allowlists
4. **CSV injection protection** -- CsvExport class and AdminBillingController sanitize cell values starting with `= + - @ \t \r`
5. **No `env()` in app code** -- All configuration access goes through `config()` helper
6. **Mass assignment protection** -- `is_admin` and `super_admin` are NOT in User `$fillable`
7. **preventLazyLoading** -- Globally enabled with production logging fallback
8. **Security headers** -- CSP with nonce, HSTS, X-Frame-Options DENY, Permissions-Policy
9. **Encrypted impersonation** -- Admin ID encrypted in session, prevents impersonating admins/self/trashed
10. **Feature flag guards** -- Route-dependent flags correctly prevent DB overrides when routes aren't registered

## Critical & High Findings

### SEC-001 [CRITICAL] -- Retention coupon endpoint missing Form Request validation

**File:** `app/Http/Controllers/Billing/SubscriptionController.php:405`

`applyRetentionCoupon()` accepts a bare `Request` instead of a Form Request. While the coupon ID comes from config, this is a POST mutation endpoint calling Stripe's `applyCoupon()`. Inconsistent with all other billing endpoints which use dedicated Form Requests.

**Fix:** Create `ApplyRetentionCouponRequest` with `authorize()` check.

---

### SEC-002 [HIGH] -- Notification API routes not feature-gated at route level

**File:** `routes/api.php:50`

Notification routes are always registered regardless of `features.notifications.enabled`. The comment says "feature-gated in controller constructor" but this exposes the endpoints when the feature is off.

**Fix:** Wrap in `if (config('features.notifications.enabled'))`.

---

### SEC-003 [HIGH] -- Webhook API routes not feature-gated at route level

**File:** `routes/api.php:67`

Same issue as SEC-002 for webhook CRUD routes.

**Fix:** Wrap in `if (config('features.webhooks.enabled'))`.

---

### SEC-004 [HIGH] -- Resume endpoint missing Form Request

**File:** `app/Http/Controllers/Billing/SubscriptionController.php:240`

`resume()` uses bare `Request` instead of a dedicated Form Request with `authorize()`.

**Fix:** Create `ResumeSubscriptionRequest`.

---

### SEC-005 [HIGH] -- Admin audit-logs/export route ordering bug

**File:** `routes/admin.php:136`

The export route `/audit-logs/export` is registered AFTER the wildcard `/audit-logs/{auditLog}` at line 139. The string "export" matches the `{auditLog}` wildcard first, causing a `ModelNotFoundException`. Other admin sections (users, feedback, contact-submissions) correctly place export before wildcard.

**Fix:** Move export route before the `{auditLog}` wildcard route.

---

### SEC-006 [HIGH] -- BillingService::getSubscriptionStatus duplicates subscription query

**File:** `app/Services/BillingService.php:192`

`getSubscriptionStatus()` loads the subscription, then calls `resolveUserTier()` which loads it again via `user->subscription('default')`. Redundant DB query.

**Fix:** Pass the already-loaded subscription to tier resolution.

---

### SEC-007 [HIGH] -- Trusted proxy reads from $_SERVER/$_ENV directly

**File:** `bootstrap/app.php:38`

Trusted proxy configuration reads `TRUSTED_PROXIES` from `$_SERVER/$_ENV` superglobals instead of `config()`. While necessary at boot time, this bypasses the config system.

**Fix:** Document the security assumption or move to a config file.
